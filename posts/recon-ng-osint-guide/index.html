🧠 What is Recon-ng?#

Recon-ng is an open-source OSINT (Open Source Intelligence) framework written in Python that helps security researchers, red teamers, and bug bounty hunters automate reconnaissance tasks.

It mimics the Metasploit interface, offering powerful modules to collect intel like domains, hosts, credentials, social profiles, leaked data, and more — all in an automated, database-backed workspace.

🎯 Key Features of Recon-ng#

🔧 Feature 🧠 Description
🧩 Modular Design 100+ modules: domains, hosts, ports, emails, creds, companies, social, etc.
🗂️ Workspaces Isolated environments for different recon jobs
🔑 API Key Integration Supports Google, Shodan, Censys, GitHub, Hunter.io, etc.
📄 Reporting Export as CSV, JSON, HTML for documentation
🧠 Automatic DB Storage Stores all data inside a local SQLite DB
🤖 Automation Support Command-line scripting, chaining, and integration with other tools
📦 Built-in Commands Like use, run, set, show, search, export, keys, etc.

💼 Real-World Use Cases#

Use Case Description
✅ Red Team OSINT Recon Identify subdomains, emails, and open ports to prepare attack surface map
🧑‍💻 Bug Bounty Hunting Discover hidden assets and exposed credentials from public sources
🛡️ Blue Team Intel Detect leaked internal data and exposed public infrastructure
📊 Reporting Auto-export reconnaissance findings for documentation or client reports

🧰 Installation on Ubuntu/Kali#

# Update system
sudo apt update && sudo apt upgrade -y

# Install dependencies
sudo apt install python3 python3-pip git -y

# Clone Recon-ng repository
git clone https://github.com/lanmaster53/recon-ng.git

# Navigate into directory
cd recon-ng

# Install Python dependencies
pip3 install -r REQUIREMENTS

✅ Run the Framework#

python3 recon-ng

⚡ Expert Tips for Recon-ng#

  • 🧠 Use search often to discover relevant modules quickly.
  • 🗂️ Use notes add to keep case notes per workspace.
  • 🔁 Combine modules: crtsh → resolve → shodan_hostname → export.
  • 💡 Integrate Recon-ng with Bash or Python for automated recon.
  • 🔒 Never forget to add valid API keys for premium modules.
  • 💬 You can script module runs using the .recon-ngrc file for automation.

3. Custom Module Development#

File Structure:

~/.recon-ng/modules/custom/
└── my_custom_module.py

Template:

from recon.core.module import BaseModule

class Module(BaseModule):
    meta = {
        'name': 'Custom Domain Scraper',
        'author': 'Your Name',
        'description': 'Fetches subdomains from internal DB',
        'query': 'SELECT DISTINCT domain FROM domains WHERE domain LIKE "%example%"',
    }

    def module_run(self, domains):
        for domain in domains:
            self.alert(f"Checking {domain}...")
            # Your logic here
            self.add_hosts(host=ip, hostname=domain)

9. Maltego Transform Server#

from maltego_trx.maltego import MaltegoMsg
from maltego_trx.transform import DiscoverableTransform

class ReconNGTransform(DiscoverableTransform):
    @classmethod
    def create_entities(cls, request: MaltegoMsg, response):
        domain = request.Value
        # Call Recon-ng modules via API
        response.addEntity("maltego.Domain", domain)
